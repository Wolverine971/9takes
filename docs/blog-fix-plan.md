---

## How to Use This Document

An agent fixing a blog should:

1. Read this entry for the person they're working on
2. Read the current draft at `src/blog/people/drafts/[Person-Name].md`
3. Read any existing YouTube transcripts listed in the "Available Research" section
4. Gather any missing transcripts listed in "Transcripts to Grab"
5. Execute the fix using the `/blog_content_creator_people` command (Update Workflow, Option 1)
6. Verify all items in the "Fix Checklist" are addressed before pushing
7. Clean up embedded reviewer feedback HTML comments (`<!-- -->`) before publication

## Agent Instructions

When tasked with fixing a specific blog:

1. **Read this entry first** — understand what's broken and what research exists
2. **Read the current draft** at `src/blog/people/drafts/[Person-Name].md`
3. **Read all available transcripts** listed in "Available Research"
4. **If transcripts need to be grabbed**, use the `/youtube-transcript` skill to download them before starting the rewrite
5. **Use the `/blog_content_creator_people` command** (Update Workflow → Option 1: Fresh Web Research) as the execution framework
6. **Verify every item in the Fix Checklist** before pushing to database
7. **Cross-reference** the quality patterns from `docs/blog-quality-review.md` ("Key Patterns: What the Best Blogs Do") against your revision
8. **Clean up reader feedback HTML comments** (`<!-- -->`) at the bottom of drafts after addressing them

**Quality bar**: The fixing agent should aim for 8.5+ on the review scale. If the revision doesn't clear that bar, iterate before pushing.
