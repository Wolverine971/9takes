So of the 6 sentinels, **4 are clean false positives** and **2 point at a real problem** (one total, one partial) — the sentinel mechanism is unreliable in both directions on this small sample.

**Separately, and more consequential:** 14 of the other 27 directories (no sentinel file present at all) are **confirmed true failures with zero detection**:

- 12 are exact-0-byte `1_create.log` orphans (keira-knightley 06-12, hailee-steinfeld 06-13, kurt-russell 06-16, nick-offerman ×2 on 06-19, john-goodman 06-20 & 06-22, susan-sarandon ×3 on 06-24/25/27, brendan-fraser 06-29, zac-efron 07-01)
- 1 died mid-stream after a real stage 1 (john-goodman 06-21, 2_fresh_eyes.log is 0 bytes)
- 1 is a total 401-auth failure across all 7 stages with real (non-empty) error text in every log (Maisy-Stella 06-20)

All 14 of these predate 2026-07-02. **The sentinel-writing code appears to have been added around 2026-07-02** — before that date, failures were completely silent (no sentinel of any kind, true or false); after that date, the sentinel fires reliably but is miscalibrated toward false positives (return code 0 is checked, not actual content/output).

(Tobey-Maguire and Rose-Blackpink also had false/mixed sentinels but aren't in the queue file at all, so their `needsReview` state can't be checked this way — consistent with the coverage-gap finding above.)

`zac-efron`'s two orphan/session-limit runs (07-01, 07-02) never got a `failed` entry either — they show up as `retryCount: 2` on a `skipped` entry with `skipReason: "draft already exists"` once the 07-02_114347 manual run finally succeeded.

---

## Bottom line

1. **Two independent bugs, opposite directions.** Before ~2026-07-02: real failures (14 confirmed) were completely silent — no sentinel, no queue record, no alert. From ~2026-07-02 onward: a `FAILED_AT_STAGE` sentinel now fires, but 4 of its 6 firings are false positives on genuinely-completed runs (checking `exit=0` instead of actual output), one is a true total failure it correctly-by-accident labeled, and one caught a real dead stage that self-healed via an unlogged recovery step.
2. **The false sentinel has a real downstream cost**: it directly sets `needsReview: true` on clean, publication-ready completions (steve-martin, jeff-goldblum), manufacturing review work that doesn't need to exist.
3. **The revision loop fires reliably but rarely closes the gap it exists for**: 5/5 sub-8.5 grades triggered revision, but only 1/5 crossed back over 8.5, one was flat, and one regressed.
4. **The 2026-07-01 grader recalibration is real and measurable**: first-pass mean dropped from 8.80 to 8.31 (letters shifted from all-B+/A to including two C's), and post-recalibration output has not produced a single A even after revision.
5. **The queue file undercounts reality**: 8 of 18 tracked people (all manual/daytime runs) aren't recorded in it at all, and its `failed` array has never been used.
