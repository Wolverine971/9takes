# Personality Series — Type-Anchored Content + Engagement Loop

**Status:** Strategy layer. Connects the content (`post-ideas/2026-05-19_personality-series-intro-arc-lineup.md`), the voice (`personality-series-north-star.md`), and the engagement map (`enneagram-engagement-clusters.md`) into one system.
**Owner:** DJ Wayne · **Adopted:** 2026-06-17

## The One Idea

**Content and engagement are the same loop.** Every personality-analysis post anchors one Enneagram type. That type has a **pond** — a cluster of people already posting about its core wound in pop-psych vernacular (perfectionism, anxiety, people-pleasing, ADHD…). The post is the **bridge** we bring to that pond.

So we stop treating "make posts" and "engage accounts" as separate chores. The week we post the Type N celebrity, we work the Type N pond — and the fresh post is the natural thing to route the strangers we meet there back to.

```
POST (anchors a type)  →  POND (that type's cluster, worked the same week)
      ↑                                   │
      │                                   ▼
   CAPTURE  ←—————————————————————  BRIDGE (pond stranger → our post → blog → 9takes)
```

## The Loop, step by step

1. **POST** — Publish the type-first carousel (see north star). Caption opens with the type's self-ID hook, uses the celebrity as proof, lands on the type's core struggle.
2. **POND** — That same week, engage the type's cluster from `enneagram-engagement-clusters.md`: comment in the rooms (hashtags + big-creator comment sections), follow 5–10 new accounts, validate the pattern first. _Comment value days before any link._
3. **BRIDGE** — A stranger in the "people-pleaser" room hits our profile/pinned comment → meets the richest live destination for their type (the celebrity post when one exists, otherwise the type page `/personality-analysis/type/N` or the type blog `enneagram-corner/enneagram-type-N`) → 9takes. They never got a cold "take the test" pitch; they got mirrored.
4. **CAPTURE** — The post's engagement bait ("Are you a Type N? Tag one") + the accounts surfaced in the pond feed back into `instagram-engagement-targets.md` (append-only) and `account-profiles/`.

## How this changes the TIDBITS (the integration you asked for)

The slide-2 tidbits are no longer trivia — **they're the mirror.** A tidbit earns its slot when it reveals the type's _wound_, because that wound is exactly what the pond is searching for. When someone from the "I abandon myself" pond sees a Type 2 slide, a tidbit should make them feel _seen_.

**Tidbit test (added to the north star's five):** Does this tidbit reveal the type's core struggle — the thing the pond is already searching for? If it only reveals the celebrity's résumé, it's trivia. Move it to "Hold for stories."

This is the Sabrina reorientation (2026-06-17) generalized: her slide went from lore (closet studio, church probe, fire-escape rats) to wound-revealing (lifelong drive, the grind nobody mentions, killing her own viral bit, the wink-as-control) — so it mirrors the imposter-syndrome / hustle-burnout pond, not just Sabrina fans.

## How this changes the CAPTION

Already handled by the type-first format (2026-06-16): open with the pond's vernacular (reader self-ID) → name the type → celebrity as proof → the struggle → blog. The first line of every caption _is_ the pond's search query. Keep it that way.

## Operating cadence

- **Per post (the week it runs):** post → work its pond (comment-first) → route warm contacts to the post → log accounts to the targets doc.
- **Match the calendar:** the existing Tue/Fri cadence in the lineup doc is the _content_ clock; the pond work runs the same week as each post, not on its own schedule.
- **Don't diagnose strangers.** Name the _pattern_, never "you're a Type X." (See [feedback: Instagram comment style].)
