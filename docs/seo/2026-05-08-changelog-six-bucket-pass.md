### 3b. Bucket 3 stretch — questions↔blog retrieval RPC

**New file:** `supabase/migrations/20260508_related_blogs_for_question.sql` (117 lines, staged)

**What it does:** RPC `get_related_blogs_for_question(question_id_input bigint, limit_count int DEFAULT 5)` that:

1. Pulls comma-separated keywords from `question_keywords.keywords` for the input question
2. Splits + normalizes into a set of trimmed lowercase terms
3. Matches against `blogs_content.tags` (text[]) AND `blogs_famous_people.keywords` (text[])
4. Returns top N rows: `slug, title, description, url, source, relevance` — relevance = tag-overlap count, ties broken alphabetically by slug

**Bug history (during today's session):**

- v1: `= ANY((SELECT term_list FROM terms_lower))` — failed with `text = text[]` operator error. Postgres parsed the scalar subquery as returning `text[]` rows.
- v2: replaced with `IN (SELECT term FROM terms)` — fixed
- v3: also referenced `p.slug` on `blogs_famous_people`. That column doesn't exist on this table. Fixed by deriving slug from `loc` via `split_part(p.loc, '/personality-analysis/', 2)` with `LIKE '%/personality-analysis/%'` guard.

**Final state:** verified all column references against `database.types.ts`. RPC is `SECURITY INVOKER` (preserves RLS). Granted to `anon`, `authenticated`, `service_role`. Header comments cross-link the SEO source docs.

**Apply with:** Supabase Studio SQL editor, or `supabase migration up` if you use the local CLI. The migration is idempotent (`CREATE OR REPLACE FUNCTION`).

**Consumer not yet wired:** `src/routes/questions/[slug]/+page.server.ts` would need to call this RPC and pass results to the page; a "Related blog posts" block component would need to be added below the comments. That work was NOT done — only the migration is staged.

### 3c. Bucket 1 follow-up — Person.sameAs backfill script

**New file:** `scripts/backfill-person-sameas.js` (448 lines, staged)

**What it does:** Backfills `same_as` (text[]), `wikidata_qid`, and `imdb_id` on `blogs_famous_people` rows. Three-source merge:

1. Reads frontmatter from `src/blog/people/drafts/*.md` (58/402 drafts already have these fields per the audit)
2. Reads `blogs_famous_people` rows via Supabase service-role client
3. For each row missing data:
   - **`from-frontmatter`**: if draft has values and DB doesn't, fill DB from draft (no API calls)
   - **`lookup-needed`**: if both empty, query Wikidata `wbsearchentities` with the person's name; resolve QID → English Wikipedia URL via `wbgetentities` sitelinks; resolve IMDb nconst via Wikidata claim P345

**Flags:**

- `--apply` (default is dry-run)
- `--limit=N` (default 100)
- `--rank=recent|traffic|alpha` (traffic uses 90-day `page_analytics_visits` aggregate; recent uses `lastmod DESC`; alpha uses `person ASC`)
- `--write-frontmatter` (sync results back to drafts so the two stay aligned)
- `--help`

**Safety:**

- Dry-run by default
- Wikidata rate-limited to 1 req / 200ms
- Polite User-Agent string (`9takes-sameAs-backfill/1.0 (https://9takes.com; djwayne35@gmail.com)`)
- Only persists Wikipedia URLs that Wikidata returned — no guessing
- Validates IMDb nconst format `^nm[0-9]+$` (matches DB CHECK constraint from migration `20260420`)
- Updates DB by `id` (PK), not by slug (which doesn't exist on this table)

**Bug history (during today's session):**

- v1: selected `slug` column from `blogs_famous_people`. Doesn't exist. Fixed: select `loc`, derive slug via `deriveSlug(loc)` helper (parses path segment after `/personality-analysis/`).
- Order-by `slug` for alpha rank also broken; fixed to order by `person`.

**Schema infrastructure already shipped (not in this script, was prior work):**

- `src/lib/utils/schema.ts` `buildPersonSameAsUrls()` — composes the sameAs array from raw inputs
- `supabase/migrations/20260420_blogs_famous_people_jsonld_fields.sql` — the DB columns

**Usage example:**

```bash
node scripts/backfill-person-sameas.js                   # dry-run, top 100 by recency
node scripts/backfill-person-sameas.js --apply --limit=10   # process 10 rows
node scripts/backfill-person-sameas.js --rank=traffic --apply --limit=50 --write-frontmatter
```
